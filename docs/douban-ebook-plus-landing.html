01 · 功能特性

7 大平台，一键直达

  1. 7 平台接入

    微信读书、豆瓣阅读、得到、多看阅读、网易蜗牛读书、Z-Library、Anna's Archive。覆盖主流中文电子书平台与外文资源库。

  2. 5 平台智能直达

    微信读书、豆瓣阅读、得到、多看、蜗牛读书支持智能解析：输入书名，自动返回书籍详情页直达链接，而非搜索结果页。

  3. Z-Library 镜像探活

    内置镜像列表，Service Worker 后台自动探活（3秒超时），返回首个可用镜像。支持用户自定义镜像、调整优先级。

  4. 可配置设置页

    独立设置页面（chrome://extensions → 扩展详情 → 扩展程序选项），支持 Z-Library 镜像增删改查、Anna's Archive 搜索地址自定义。

  5. 零依赖纯 JS

    无构建步骤，无 npm 依赖，原生 JavaScript 实现。Manifest V3 标准，现代 Chrome 扩展架构。

  6. 内联 SVG 图标

    所有平台图标通过 JS 动态生成 SVG data URI，无需网络请求，零外部依赖，离线可用。

02 · 设计理念

极简、开放、可靠

  1. 01 极简不添加多余 UI，只在豆瓣页面注入一个面板，不打扰阅读体验。
  2. 02 隐私优先不收集任何用户数据，所有查询在本地完成，零服务器依赖。
  3. 03 开放源码MIT 许可证，代码完全公开。欢迎 Fork、PR、二次开发。
  4. 04 长期维护Manifest V3 标准，基于现代 Chrome 扩展架构，可持续迭代。

04 · 常见问题

FAQ

支持哪些浏览器？
目前仅支持 Chrome 88+（Manifest V3 标准）。Edge、Arc 等基于 Chromium 的浏览器理论兼容，但未充分测试。
为什么 Z-Library 链接有时无法访问？
扩展会自动探测镜像可用性，但网络环境差异可能导致探测失败。建议前往设置页添加您所在地区可用的镜像地址。
会收集我的阅读记录吗？
不会。扩展仅在豆瓣读书页面提取书名和 ISBN，用于生成各平台链接。所有数据在本地处理，不上传任何服务器。
为什么某些平台只能跳转到搜索页？
Anna's Archive 目前仅支持搜索模式（因该平台无公开 API）。Z-Library 当镜像不可达时会降级为搜索页。
